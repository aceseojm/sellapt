# 메디스파크 랜딩페이지 퍼널 마케팅 기획서

## 1. 기획 목적

본 기획서는 `브레인시티 메디스파크 로제비앙 모아엘가` 분양 홍보 랜딩페이지를 단순 안내형이 아니라 `후킹형 전환 페이지`로 업그레이드하기 위한 실전 전략 문서다.

이번 수정 방향은 아래 2가지를 반영했다.

- 대표번호를 `010-6689-2348`로 통일
- 참조 페이지 `[m-bee 브레인시티 페이지](https://m-bee.kr/braincity)`의 강한 전환 구조를 벤치마킹하되, 메디스파크의 차별점이 더 강하게 터지도록 상위 버전으로 재설계

또한 랜딩페이지 개발 및 고객 DB 수집 운영 기준도 아래처럼 명확히 지정한다.

- 개발자 지정: `서정민`
- 고객 DB 저장 방식: `Google Apps Script + Google Sheets`
- 고객 DB 기본 저장 항목: `이름`, `연락처`, `개발자 정보`

## 2. 참조 페이지에서 반드시 가져와야 할 장점

참조 페이지는 정보량보다 `즉시 행동`에 집중되어 있었다. 특히 아래 요소가 강점이다.

- 첫 화면에서 바로 상담/예약 CTA 노출
- 복잡한 설명보다 짧은 핵심 문장 중심
- 사용자가 고민하기 전에 폼으로 유도하는 구조
- 모바일 중심의 짧고 빠른 스크롤 흐름
- 방문자가 "나만 늦는 것 아닐까"를 느끼게 하는 전환형 분위기

메디스파크 랜딩페이지는 여기서 한 단계 더 나아가야 한다.

- 단순 문의 유도가 아니라 `왜 지금 바로 연락해야 하는지`를 만들어야 한다
- 누구라도 멈춰 보게 만드는 강한 후킹 문장이 필요하다
- 실거주자와 투자자 모두 자기 이야기처럼 느끼게 해야 한다
- CTA가 페이지 곳곳에서 반복적으로 작동해야 한다

## 3. 메디스파크에서 실제로 가장 강한 후킹 포인트

현재 보유 자료를 기준으로 보면, 이 현장의 본질은 단순히 "브레인시티 신축 아파트"가 아니다.

실제로 강한 포인트는 아래 4개다.

1. `학교, 병원, 상업시설, 공원을 도보로 누리는 생활 입지`
2. `브레인시티 의료복합타운과 종합병원 개발 기대감`
3. `84A, 84B, 101 중심의 선호도 높은 가족형 평면`
4. `브레인시티 안에서도 희소한 생활권 완성형 입지`

즉, 이 페이지는 "좋은 아파트 소개"가 아니라 아래처럼 인식되어야 한다.

`살기 좋은데, 미래가치까지 보이고, 지금 선점하지 않으면 놓칠 수 있는 자리`

`살기 좋아서 끌리고, 오를 이유가 보여서 더 빨리 잡고 싶은 자리`

## 5. 이번 랜딩페이지의 목표 전환 행동

이 랜딩페이지는 아래 중 하나를 반드시 일으켜야 한다.

- 관심고객 등록
- 전화 문의
- 카카오/문자 상담
- 방문 예약
- 평면/분양일정 자료 요청

대표번호는 모든 CTA와 고정 상담영역에 아래와 같이 통일한다.

- `대표번호 010-6689-2348`

### 상담 버튼 동작 방식 필수 변경

기존처럼 `상담하기 클릭 → 즉시 전화걸기`만 되면 이탈이 많아질 수 있다.  
권장 흐름은 아래와 같다.

1. 사용자가 `상담하기` 클릭
2. 간단한 레이어 팝업 또는 별도 상담 폼 오픈
3. `이름`, `연락처`만 짧게 입력
4. `상담 연결받기` 버튼 클릭
5. 접수 완료 후 `빠르게 연락드리겠습니다` 안내
6. 동시에 `전화 바로걸기 010-6689-2348` 버튼도 함께 노출

즉, `정보 남기고 상담받기`와 `바로 전화걸기`를 동시에 제공하되, 메인 전환은 `이름 + 연락처 접수형`으로 설계한다.

### DB 저장 시 필수 기록 항목

상담 접수 시 아래 항목은 반드시 함께 저장되도록 한다.

- 고객 이름
- 고객 연락처
- 개발자 정보: `서정민`

즉, 사용자가 입력하는 값은 `이름`, `연락처` 2개지만, 시스템에서는 고정값으로 `개발자=서정민`을 함께 저장해야 한다.

## 10. 폼 설계 원칙

입력은 최대한 짧게 가져간다.

### 필수 입력

- 이름
- 연락처
- 관심 평형

### 선택 입력

- 실거주 / 투자 관심
- 방문 상담 희망 여부

### 폼 제목 추천

- `1분 등록으로 핵심자료 받기`
- `관심고객 등록하고 빠르게 안내받기`

### 상담 전용 미니폼 권장 구조

- 입력항목 1: `성함`
- 입력항목 2: `연락처`
- CTA 버튼: `상담 연결받기`
- 보조버튼: `바로 전화하기 010-6689-2348`

### 시스템 내부 전송 항목

사용자 화면에는 최소 항목만 보이되, 실제 전송 데이터에는 아래 항목이 포함되어야 한다.

- `name`
- `phone`
- `developer`

여기서 `developer` 값은 사용자가 입력하지 않고 고정값으로 아래처럼 전송한다.

- `developer: 서정민`

### 거부감 줄이는 안내 멘트 추천

이 부분은 아주 중요하다. 사용자가 이름과 연락처를 남길 때 가장 싫어하는 감정은 `귀찮다`, `영업전화 많이 오겠다`, `개인정보가 부담된다`는 점이다.  
그래서 문구는 짧고 안심되게 설계해야 한다.

#### 최우선 추천 멘트

- `이름과 연락처만 남겨주시면 확인 후 빠르게 안내드립니다.`
- `불필요한 반복 연락 없이 요청하신 내용만 안내해드립니다.`
- `간단한 정보만 남기시면 상담 가능한 시간에 맞춰 연락드립니다.`

#### 부드러운 안심형 멘트

- `복잡한 입력 없이 성함과 연락처만 남겨주세요.`
- `관심 타입, 분양 일정, 위치 안내만 빠르게 도와드립니다.`
- `상담 요청 후 담당자가 순차적으로 확인해 연락드립니다.`

#### 개인정보 부담 완화 멘트

- `입력하신 정보는 상담 안내 용도로만 사용됩니다.`
- `최소 정보만 간단히 남겨주시면 됩니다.`
- `원치 않으시면 언제든 상담 중단 요청이 가능합니다.`

### 실제 추천 조합

가장 무난하고 전환율 좋은 조합은 아래다.

- 제목: `간단히 남겨주시면 빠르게 연락드립니다`
- 안내문구: `성함과 연락처만 남겨주시면 요청하신 내용만 빠르게 안내드립니다.`
- 입력창: `성함`, `연락처`
- 버튼: `상담 연결받기`
- 하단보조문구: `입력하신 정보는 상담 안내 용도로만 사용됩니다.`

### 피해야 할 문구

- `필수입력 후 상담 가능`
- `정확한 상담을 위해 자세히 작성해주세요`
- `개인정보 수집에 동의해야 합니다`만 크게 노출하는 방식
- `지금 입력하지 않으면 혜택 종료`처럼 과도한 압박형 문구

거부감을 줄이려면 압박보다 `간단함`, `빠름`, `안심`을 먼저 느끼게 해야 한다.

## 10-1. 고객 DB 수집 및 저장 구조

이 랜딩페이지는 단순 폼 제출이 아니라 `고객 DB 축적`까지 고려해서 설계해야 한다.

### 운영 목적

- 상담 신청자 정보를 누락 없이 저장
- 광고/페이지별 유입 고객을 체계적으로 관리
- 추후 상담 이력 정리와 성과 분석에 활용

### 저장 방식

- 프론트엔드 폼 제출
- Google Apps Script 웹앱으로 데이터 전송
- Google Sheets에 자동 저장

### 고정 운영 정보

- 개발자명: `서정민`

### 구글시트 필수 컬럼

최소한 아래 3개 컬럼은 반드시 저장한다.

- `이름`
- `연락처`
- `개발자`

권장 컬럼은 아래처럼 확장한다.

- `접수일시`
- `이름`
- `연락처`
- `개발자`
- `유입페이지`
- `상담구분`
- `비고`

### 현재 요청 기준 최소 저장 스펙

이번 요청 기준으로는 아래가 반드시 저장되면 된다.

- 이름
- 연락처
- 개발자 정보 `서정민`

### Apps Script 연동 전제 문구

개발 명세에는 아래 내용을 포함하는 것이 좋다.

- 랜딩페이지의 상담 폼은 Google Apps Script 웹앱 URL로 POST 전송한다
- 시트 저장 시 `개발자` 항목에 `서정민`을 고정 입력한다
- 고객 입력값은 `이름`, `연락처`만 받고 저장 시 함께 기록한다

### 개발자 정보 기록 목적

개발자 정보를 함께 남기는 이유는 아래와 같다.

- 페이지 제작 담당자 식별
- 다중 랜딩페이지 운영 시 관리 편의성 확보
- DB 출처 관리 및 유지보수 편의성 확보

### 실무 명시 문구

`본 랜딩페이지의 고객 DB는 Google Apps Script를 통해 Google Sheets에 저장되며, 저장 항목에는 이름, 연락처, 개발자 정보(서정민)가 포함된다.`

## 14. 결론

메디스파크 랜딩페이지는 예쁘게 만드는 것이 목적이 아니다. `누가 봐도 한번 멈추고, 한번 읽고, 한번 문의하게 만드는 구조`가 목적이다.

성공 확률을 높이려면 아래 순서를 흔들리면 안 된다.

1. `첫 화면에서 강하게 낚는다`
2. `생활 입지로 공감시킨다`
3. `미래가치로 기대감을 키운다`
4. `평면으로 실거주 확신을 준다`
5. `자료 제공형 CTA로 바로 전환시킨다`

메디스파크는 후킹 포인트가 충분하다. 중요한 것은 정보를 많이 쓰는 것이 아니라, 사람들이 `지금 안 보면 손해일 수 있겠다`고 느끼게 만드는 순서와 문장이다.
